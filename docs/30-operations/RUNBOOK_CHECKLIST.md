# Runbook Checklist

## 실행 체크
- [ ] DB 파일 존재 확인 (`data/dashboard.db`)
- [ ] 스키마 최신 반영 (`sql/schema.sql`)
- [ ] 서버 기동 확인 (`http://127.0.0.1:8765/api/health`)
- [ ] 최신 일자 자동 로딩 확인

## 배포 전 체크
- [ ] KPI 카드 클릭 -> 상세 페이지 이동 확인
- [ ] 상세 페이지 추이/집계근거/가이드 렌더 확인
- [ ] 운영 분석 패널 수치 갱신 확인
- [ ] 모바일 뷰(<=700px) 레이아웃 확인

## 장애 대응
1. 접속 불가
- 서버 프로세스 재기동
- 포트 점유 확인

2. 데이터 없음
- `kpi/daily` 응답 확인
- 이벤트 적재 스크립트 재실행

3. 수치 이상
- `order_events` 원천행 확인
- `v_daily_kpis` 계산식 검증
